Project Overview
 
Project Name: Retail Inventory Management System (RIMS)

Project Type: Technical Business Analyst Portfolio Project

Objective: 

* Enhance visibility across all warehouses and stores.

* Reduce discrepancies between recorded and actual data.

* Turn inventory and sales data into practical insights to help guide supply-chain decisions. 

Business Problem:

* Inventory mismatches lead to stock-outs and overstock.

* Lack of a centralized dashboard for the warehouse and supply-chain team.

* Delayed or inaccurate reporting affects decision-making.

Functional Requirements 

* Track inventory in real-time across various warehouses and stores.

* Generate automated low-stock and overstock alerts.

* Provide Tableau dashboards for executive insights.

* Export inventory data to Excel for ad-hoc reporting.

* Role-based access for stakeholders.
 

Non-Functional Requirements

* Daily data refresh for dashboards.

* Support 1000+ SKUs.

* Web-based interface for dashboards.

* Reports should load within 5 minutes.

* Data accuracy >=95%.
