Project Overview

This project simulates a real-world Retail Inventory Management System designed to optimize stock levels, reduce inventory shrinkage, and improve reporting accuracy.

This project demonstrates end-to-end Business Analysis activities, including:

1. Business Requirement Documentation (BRD)
2. Functional Requirement Documentation (FRD)
3. Jira user Stories
4. SQL Data Analysis
5. Dashboard Reporting (Tableau)

Business problem

Retail companies often face:
1. Overstocking and stockouts
2. Inaccurate inventory tracking
3. Poor demand visibility
4. Manual reporting inefficiencies

This project provides structured documentation and analytical solutions to address these issues.
